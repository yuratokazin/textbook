### Разработка дополнительных общеразвивающих программ

**Методические рекомендации по разработке дополнительных общеразвивающих программ**, в том числе в части интеграции с учебными предметами «Труд (технология)», «Музыка», «Изобразительное искусство», «Физическая культура»



Методические рекомендациинаправлены на синхронизацию дополнительных общеразвивающих программ (ДОП) с  основным общим образованием, обеспечивая единство воспитательного и  образовательного пространств. **Особенности интеграции по предметным областям:**

Практическое задание: «Создание страницы-визитки „Мои увлечения“»

**Цель работы:** Продемонстрировать навыки работы с HTML-структурой, стилизацией CSS и  применением технологии Flexbox для создания адаптивной страницы.

### 1. Техническое задание (ТЗ)

Вам необходимо создать веб-страницу, состоящую из трех основных смысловых блоков:

1. **Шапка (Header):** Логотип (ваше имя) и навигационное меню (ссылки-якоря).
2. **Основная часть (Main):**
   - Краткое приветствие и ваше фото.
   - Блок «Мои навыки/хобби» (не менее 3-х карточек, выровненных в ряд с помощью Flexbox).
3. **Подвал (Footer):** Иконки социальных сетей или контактные данные.

### 2. Требования к реализации

- **HTML:**
  - Использовать семантические теги: `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`.
  - Все изображения должны иметь атрибут `alt` с описанием.
  - Код должен быть отформатирован (соблюдена вложенность).
- **CSS:**
  - Стили вынесены в отдельный файл `style.css`.
  - Задана цветовая гамма (минимум 3 цвета: фон, основной текст, акцентный цвет для кнопок/ссылок).
  - Использован Google Fonts или системный шрифт без засечек (Arial, Verdana).
  - Карточки увлечений должны иметь скругление углов (`border-radius`) и небольшую тень (`box-shadow`).
- **Адаптивность:**
  - При уменьшении ширины экрана до 600px карточки увлечений должны перестраиваться из ряда в колонку.

### 3. Этапы выполнения

1. **Проектирование:** Набросайте схему страницы на бумаге.
2. **Верстка:** Создайте каркас в `index.html`.
3. **Оформление:** Настройте внешний вид в `style.css`.
4. **Проверка:** Откройте файл в браузере и проверьте работу ссылок и отображение на мобильном устройстве (F12 -> режим адаптивности).

### 4. Критерии «Пятерки»

- Код проходит проверку без ошибок.
- Страница выглядит аккуратно, текст легко читается.
- Добавлен хотя бы один интерактивный элемент (например, изменение цвета кнопки при наведении — `:hover`).

---



Для эффективного старта в веб-разработке рекомендую установить этот «базовый набор» плагинов.  Они превратят VS Code из обычного блокнота в мощную среду разработки,  которая сама подсказывает ошибки.

### 1. Обязательные (для комфортной работы)

- **Live Server** — Самый важный плагин. Позволяет запустить локальный сервер одной кнопкой. Страница в браузере будет **автоматически обновляться** при каждом сохранении кода (Ctrl+S). Больше не нужно нажимать F5.
- **Russian Language Pack** — Русификатор интерфейса VS Code, если ученикам сложно ориентироваться в английском меню.

### 2. Для чистоты и красоты кода

- **Prettier - Code formatter** — «Магическая» кнопка для наведения порядка. Одним нажатием (или при  сохранении) выравнивает все отступы, расставляет кавычки и делает код  читаемым. Приучает детей к профессиональным стандартам оформления.
- **Auto Close Tag** — Автоматически добавляет закрывающий тег. Если ученик пишет `<div>`, плагин сам допишет `</div>`.
- **Auto Rename Tag** — Если изменить имя открывающего тега (например, `h1` на `h2`), плагин автоматически изменит и закрывающий.

### 3. Визуальные помощники (снижают число ошибок)

- **vscode-icons** — Заменяет стандартные значки файлов в левой панели на красивые иконки  (HTML, CSS, JS). Помогает быстрее ориентироваться в структуре проекта.
- **Color Highlight** — Если в CSS написать код цвета (например, `#ff0000`), плагин подсветит это слово соответствующим цветом прямо в редакторе.
- **Indent-rainbow** — Раскрашивает вертикальные отступы в разные цвета. С ним сразу видно,  какой блок кода к какому уровню относится, что критично для HTML.

### 4. Для продвинутых (ускорение)

- **Emmet** (встроен в VS Code по умолчанию) — Научите детей писать `! + Tab` для создания структуры документа или `div.container>ul>li*3` для создания целого списка одной строкой. Это вызывает настоящий восторг у новичков.

  - *Развивающие:* развить алгоритмическое мышление, навыки проектирования интерфейсов и командной работы.

**5. Организационно-педагогические условия**

- **Формы обучения:** очная с использованием дистанционных технологий (для работы над кодом дома).
